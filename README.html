🔍 What Is SSH & How Does It Work?

The Problem It Solves

When you push code to GitHub, GitHub needs to verify who you are. Previously, you could use your password. But in 2021, GitHub disabled password authentication for Git operations because passwords are easily guessed, phished, or leaked.

How SSH Works (Simplified)

SSH uses public-key cryptography — two mathematically linked keys:

  • 🔒 Private Key (github_rsa) — Stays on your computer. Never shared. Like a physical key to your house.
  • 🔓 Public Key (github_rsa.pub) — You give this to GitHub. Like a lock that only your private key can open.

When you git push, your computer uses the private key to sign a message. GitHub checks it against the public key you uploaded. If they match → authenticated. No password needed, ever.

💡 Analogy: Think of the public key as a padlock you give to GitHub. The private key is the only key that opens that padlock. When you push code, you prove you have the key — without ever showing it.

1 Add the Public Key to GitHub

  1. Copy your public key — the long string starting with ssh-ed25519 shown above.
  2. Go to GitHub SSH settings:
    https://github.com/settings/keys
  3. Click the green "New SSH Key" button.
  4. Title: Enter a descriptive name like Laptop - Ghazi Khutouba
  5. Key type: Leave as "Authentication Key"
  6. Key: Paste the entire public key (ssh-ed25519 AAA...)
  7. Click "Add SSH Key"

Verify It Worked

3 Clone This Repo on Another Computer

If you want to work on this project from a different machine:

Option A: Copy the SSH key (easier)

  1. Copy the .ssh/ folder from this project to the new computer
  2. Place it in the same project folder on the new machine
  3. Add the private key to your SSH agent:
    $ ssh-add .ssh/github_rsa
  4. Clone normally:
    $ git clone git@github.com:ghaziomairi93/G-B.git

Option B: Generate a new key (safer)

  1. Generate a fresh key pair on the new machine:
    $ ssh-keygen -t ed25519 -C "new-computer@github"
  2. Add the new public key to GitHub (same as Step 1 above)
  3. Clone the repo

🗃 Managing Multiple SSH Keys

If you have multiple GitHub accounts or repos with different keys, use an SSH config file:

# File: ~/.ssh/config

Host github-ghazi
    HostName github.com
    User git
    IdentityFile ~/path/to/project/.ssh/github_rsa
    IdentitiesOnly yes

Then use the custom host in your remote:

$ git remote set-url origin git@github-ghazi:ghaziomairi93/G-B.git

🔐 Adding a Passphrase to the Private Key

For extra security, you can add a password (passphrase) to the private key:

$ ssh-keygen -p -f .ssh/github_rsa

You'll be prompted to enter (and confirm) a passphrase. After this, every push/pull will ask for that passphrase. You can cache it in memory with:

$ ssh-add .ssh/github_rsa

🛠 Troubleshooting

❌ "Permission denied (publickey)"

  • Did you add the public key to GitHub? Go to Settings → SSH Keys
  • Is the key format correct? It must start with ssh-ed25519
  • Test your connection: ssh -Tv git@github.com (verbose mode shows details)

❌ "Host key verification failed"

  • This means your computer doesn't recognize GitHub's server
  • Run: ssh-keyscan github.com >> ~/.ssh/known_hosts
  • Then try again

❌ "Bad permissions" on the private key

  • SSH requires the private key to be readable ONLY by you
  • On Mac/Linux: chmod 600 .ssh/github_rsa
  • On Windows (PowerShell as Admin): Fix the file permissions in Properties → Security

❌ Git still asks for username/password

  • Check that the remote is using SSH, not HTTPS:
  • git remote -v — should show git@github.com:...
  • If it shows https://..., change it:
    $ git remote set-url origin git@github.com:ghaziomairi93/G-B.git

❌ SSH agent not running (Windows)

  • Open PowerShell as Administrator and run:
    PS> Get-Service ssh-agent | Set-Service -StartupType Automatic
    PS> Start-Service ssh-agent

🛡 Security Best Practices

🚨 Critical: The github_rsa file is your digital identity on GitHub. Anyone with this file can push code as you, access your repos, and potentially do damage.

Do's ✅

  • Keep this repository private if the .ssh/ folder is included
  • Use a passphrase on the private key (see Adding a Passphrase)
  • Revoke the key on GitHub immediately if you suspect it was leaked
  • Use separate keys for different projects or machines
  • Periodically check your GitHub SSH keys at Settings → SSH Keys

Don'ts ❌

  • Never commit the private key to a public repository
  • Never share the private key via email, chat, or cloud drives
  • Don't use the same key for personal and work projects
  • Don't leave the key on shared or public computers
